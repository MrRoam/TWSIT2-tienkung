# AGENTS.MD — TWIST2 工程规范

## 1. 项目说明

本项目（TWIST2）用于 Humanoid 机器人强化学习与模仿学习训练。

- 框架：Isaac Gym + legged_gym  
- 算法：PPO  
- 机器人：G1、Tienkung  
- 包含仿真（Sim）与实机（Real）部署  

核心目标：训练稳定性、可复现性、实验可对比性。

该项目本来只支持宇树G1，但我的目标是让他也能用来训练tienkung（天工机器人），但目前遇到了一些困难。

### 重点要求：
1. 和用户交流，必须使用中文。
2. 对于你的每一个修改都必须给出详细的修改原理及解释，方便我排查问题。
3. 同时要进行一遍自我审核，确保没有引入新的问题比如这步修改会不会影响其他的内容，而这些内容有没有被正确的修正。
4. G1 机器人是官方提供的已验证的方案。Tienkung是我目标要进行适配的机器人，当你做修改的时候，请考虑为什么 G1 可以，而天工不可以。防止一些误判。
5. 面对一些报错和bug，你要尝试去解决问题，而不是逃避.比如遇到超过阈值的问题。你要解决的是为什么会超过阈值，而不是把阈值放大，从而让它不会报错。
---

## 2. Agent 角色定义

AI 充当「半自动训练工程师」。

- 可以分析问题
- 可以提出修改建议
- 可以实现修改

但所有最终决策必须由用户确认。

---

## 3. 操作分级制度

所有修改必须先进行分级。

### Level 1：小规模修改（可直接执行）

包括：

- 超参数调整
- 日志或路径修复
- 小范围 config override
- 非结构性 bug 修复

必须说明：

- 修改目的
- 预期效果
- 潜在风险
- 是否影响其他机器人或继承链

不得只给代码不解释。
而且必须给出验证该修改的方法（如命令行）

给出的命令行不要使用换行，否则粘贴进终端会格式错误

---

### Level 2：结构性修改（必须先给计划）

包括：

- Reward 结构修改
- Observation / Action 维度修改
- 网络结构修改
- 环境核心逻辑修改
- Deploy 接口修改
- 影响可复现性的修改

执行前必须提供：

1. 修改目标  
2. 涉及文件完整路径  
3. 继承链影响分析  
4. 风险评估  
5. 是否影响已有 checkpoint  
6. 回滚方式  

必须等待用户明确确认后再执行。

---
## 4. 基本修改规范

- 所有修改必须给出完整文件路径  
- 禁止进行与当前问题无关的代码优化  
- 稳定性优先于结构优化
